IfTagHandler Tests

Tests the IfTagHandler

TagHandlers are called twice on AJAX: During RESTORE_VIEW and RENDER_RESPONSE. It is possible that the condition changes between those phases; this causes the component tree to change.

Example: A button adds an item to a list with ajax and updates the if-tag, which tests for even entries in the list.

  • During phase 1, the nr of entries is let's say even; an outputText is added with the text "size is even".
  • During phase 5, an item is added to the list.
  • During phase 6, the IfTagHandler is called again, now evaluating to "odd", in which case a different control is added.
Result: Only the "odd" control is rendered.
